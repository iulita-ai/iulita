# Конфигурация

Iulita использует многослойную систему конфигурации, которая поддерживает установку без конфигурации для локального использования и полную настройку для продвинутых развёртываний.

## Слои конфигурации

Конфигурация загружается последовательно, каждый следующий слой переопределяет предыдущий:

```
1. Скомпилированные значения по умолчанию (всегда присутствуют)
2. TOML-файл (~/.config/iulita/config.toml, необязателен)
3. Переменные окружения (префикс IULITA_*)
4. Секреты из хранилища ключей (macOS Keychain, Linux SecretService)
5. Переопределения в БД (таблица config_overrides, редактируемые в рантайме)
```

### Слой 1: Скомпилированные значения по умолчанию

`DefaultConfig()` предоставляет рабочую конфигурацию без внешних файлов. Все ID моделей, тайм-ауты, настройки памяти и флаги функций имеют разумные значения. Система работает из коробки с одним API-ключом.

### Слой 2: TOML-файл

Необязателен. Расположен в `~/.config/iulita/config.toml` (или `$IULITA_HOME/config.toml`).

TOML-файл **пропускается**, если:
- Файл не существует по пути конфигурации
- Существует файл-маркер `db_managed` (режим веб-мастера)

Полный справочник см. в `config.toml.example`.

### Слой 3: Переменные окружения

Все настройки можно переопределить через переменные окружения `IULITA_*`:

```
IULITA_CLAUDE_API_KEY      → claude.api_key
IULITA_TELEGRAM_TOKEN      → telegram.token
IULITA_CLAUDE_MODEL        → claude.model
IULITA_STORAGE_PATH        → storage.path
IULITA_SERVER_ADDRESS      → server.address
IULITA_PROXY_URL           → proxy.url
```

**Правило маппинга**: убрать префикс `IULITA_`, привести к нижнему регистру, заменить `_` на `.`.

### Слой 4: Секреты в хранилище ключей

Секреты хранятся в системном хранилище ключей ОС:

| Секрет | Переменная окружения | Аккаунт в хранилище |
|--------|---------------------|---------------------|
| API-ключ Claude | `IULITA_CLAUDE_API_KEY` | `claude-api-key` |
| Токен Telegram | `IULITA_TELEGRAM_TOKEN` | `telegram-token` |
| Секрет JWT | `IULITA_JWT_SECRET` | `jwt-secret` |
| Ключ шифрования конфигурации | `IULITA_CONFIG_KEY` | `config-encryption-key` |

**Цепочка резервов** для каждого секрета: переменная окружения -> хранилище ключей -> файл (только для ключа шифрования) -> автогенерация (только для JWT).

Хранилище ключей использует `zalando/go-keyring`:
- **macOS**: Keychain
- **Linux**: SecretService (GNOME Keyring, KDE Wallet)
- **Резерв**: зашифрованный файл `~/.config/iulita/encryption.key`

### Слой 5: Переопределения в БД (Config Store)

Конфигурация, редактируемая в рантайме, хранится в таблице `config_overrides` SQLite. Управляется через:
- Редактор конфигурации дашборда
- Инструмент `skills` в чате (действие `set_config`)
- Веб-мастер настройки

**Возможности:**
- Шифрование AES-256-GCM для секретных значений
- Мгновенная горячая перезагрузка через шину событий
- Аудит-логирование (кто что изменил, когда)
- Защита неизменяемых ключей, требующих перезапуска

## XDG-совместимые пути

| Платформа | Конфигурация | Данные | Кэш | Состояние |
|-----------|-------------|--------|-----|-----------|
| **macOS** | `~/Library/Application Support/iulita/` | `~/Library/Application Support/iulita/` | `~/Library/Caches/iulita/` | `~/Library/Application Support/iulita/` |
| **Linux** | `~/.config/iulita/` | `~/.local/share/iulita/` | `~/.cache/iulita/` | `~/.local/state/iulita/` |

**Переопределение**: задайте `IULITA_HOME` для использования пользовательского корня с поддиректориями `data/`, `cache/`, `state/`.

### Производные пути

| Путь | Расположение |
|------|-------------|
| Файл конфигурации | `{ConfigDir}/config.toml` |
| База данных | `{DataDir}/iulita.db` |
| ONNX-модели | `{DataDir}/models/` |
| Навыки | `{DataDir}/skills/` |
| Внешние навыки | `{DataDir}/external-skills/` |
| Файл логов | `{StateDir}/iulita.log` |
| Ключ шифрования | `{ConfigDir}/encryption.key` |

## Мастер настройки

### CLI-мастер (`iulita init`)

Интерактивная настройка, которая проводит через:
1. Выбор LLM-провайдера (Claude/OpenAI/Ollama, множественный выбор)
2. Ввод API-ключа (сохраняется в хранилище ключей)
3. Необязательные интеграции (Telegram, прокси, эмбеддинги)
4. Выбор модели (динамическая загрузка от провайдера)

Секреты сохраняются в хранилище ключей; остальное — в `config.toml`.

### Веб-мастер настройки (Docker)

Для Docker-развёртываний без терминального доступа:

1. Сервер стартует в **режиме настройки**, когда LLM не сконфигурирован и мастер не завершён
2. Только режим дашборда (без навыков, планировщика или каналов)
3. 5-шаговый мастер: Приветствие/Импорт -> Провайдер -> Конфигурация -> Функции -> Завершение
4. Поддержка импорта TOML (вставка существующей конфигурации)
5. Создание файла-маркера `db_managed` (отключает загрузку TOML)
6. Установка `_system.wizard_completed` в config_overrides

## Горячая перезагрузка

Эти настройки можно менять во время выполнения без перезапуска:

| Настройка | Триггер | Механизм |
|-----------|---------|----------|
| Модель/токены/ключ Claude | Редактор конфигурации дашборда | `UpdateModel()`/`UpdateMaxTokens()`/`UpdateAPIKey()` |
| Токен Telegram | Редактор конфигурации дашборда | `channelmgr.UpdateConfigToken()` -> перезапуск экземпляра |
| Включение/отключение навыков | Дашборд или чат | `registry.EnableSkill()`/`DisableSkill()` |
| Конфигурация навыков (API-ключи) | Редактор конфигурации дашборда | `ConfigReloadable.OnConfigChanged()` |
| Системный промпт | Редактор конфигурации дашборда | `asst.SetSystemPrompt()` |
| Бюджет мышления | Редактор конфигурации дашборда | `asst.SetThinkingBudget()` |

### Настройки, требующие перезапуска

Эти требуют полного перезапуска:
- `storage.path`
- `server.address`
- `proxy.url`
- `security.config_key_env`

## Шифрование AES-256-GCM

Секретные значения конфигурации в БД шифруются:

1. **Источник ключа**: переменная `IULITA_CONFIG_KEY` -> хранилище ключей -> автогенерируемый файл
2. **Алгоритм**: AES-256-GCM (аутентифицированное шифрование)
3. **Формат**: `base64(12-байт-nonce ‖ шифротекст)`
4. **Авто-шифрование**: ключи, объявленные как `secret_keys` в SKILL.md, всегда шифруются
5. **Без утечки**: API дашборда возвращает пустые значения для зашифрованных ключей
