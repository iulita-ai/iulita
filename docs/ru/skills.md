# Навыки

Навыки — это инструменты, которые ассистент может вызывать во время разговоров. Каждый навык предоставляет один или несколько инструментов для LLM с именем, описанием и JSON-схемой входных данных.

## Интерфейс навыка

```go
type Skill interface {
    Name() string
    Description() string
    InputSchema() json.RawMessage  // nil для текстовых навыков
    Execute(ctx context.Context, input json.RawMessage) (string, error)
}
```

**Необязательные интерфейсы:**
- `CapabilityAware` — `RequiredCapabilities() []string`: навык исключается, если какая-либо возможность отсутствует
- `ConfigReloadable` — `OnConfigChanged(key, value string)`: вызывается при изменении конфигурации во время выполнения
- `ApprovalDeclarer` — `ApprovalLevel() ApprovalLevel`: требование подтверждения

## Уровни подтверждения

| Уровень | Поведение | Используется |
|---------|----------|-------------|
| `ApprovalAuto` | Немедленное выполнение (по умолчанию) | Большинство навыков |
| `ApprovalPrompt` | Пользователь должен подтвердить в чате | Docker executor |
| `ApprovalManual` | Администратор должен подтвердить | Shell exec |

Процесс подтверждения **неблокирующий**: навык возвращает "awaiting approval" в LLM. Следующее сообщение пользователя проверяется по словарю подтверждения с учётом локали (да/нет на 6 языках).

## Встроенные навыки

### Группа Память

| Инструмент | Входные данные | Описание |
|-----------|---------------|----------|
| `remember` | `content` | Сохранить факт. Проверяет дубликаты через FTS. Запускает авто-эмбеддинг. |
| `recall` | `query`, `limit` | Поиск фактов через FTS5. Применяет временное затухание + MMR-переранжирование. Усиливает извлечённые факты. |
| `forget` | `id` | Удалить факт по ID. Каскадно удаляет из FTS и таблиц векторов. |

Подробности см. в [Память и инсайты](memory-and-insights.md).

### Группа Инсайты

| Инструмент | Входные данные | Описание |

| `dismiss_insight` | `id` | Удалить инсайт |
| `promote_insight` | `id` | Продлить или снять срок действия инсайта |

### Веб-поиск и загрузка

| Инструмент | Входные данные | Описание |
|-----------|---------------|----------|
| `websearch` | `query`, `count` | Веб-поиск через Brave API + DuckDuckGo (резерв). 1-10 результатов. |

Цепочка веб-поиска: `Brave → DuckDuckGo` через `FallbackSearcher`. DuckDuckGo не требует API-ключа, поэтому веб-поиск всегда работает.

| Инструмент | Входные данные | Описание |

## Текстовые навыки (внедрение в системный промпт)

Навыки могут представлять собой только внедрение в системный промпт — без метода `Execute`, без определения инструмента для LLM.

### Формат SKILL.md

```yaml
---
name: my-skill
description: What this skill does
capabilities: [optional-cap]
config_keys: [skills.my-skill.setting]
secret_keys: [skills.my-skill.api_key]
force_triggers: [keyword1, keyword2]
---

Markdown instructions injected into the system prompt.
```

- Навыки с `InputSchema() == nil` влияют только на `staticSystemPrompt()`
- Тело markdown становится частью инструкций LLM
- `force_triggers` принудительно вызывают конкретные инструменты при совпадении ключевых слов с сообщением пользователя

### Пути загрузки

1. **Встроенные в Go-пакеты**: `//go:embed SKILL.md` + `LoadManifestFromFS()`
2. **Внешняя директория**: `LoadExternalManifests(dir)` из `~/.local/share/iulita/skills/`
3. **Установленные внешние**: через маркетплейс ClawhHub или URL

## Менеджер навыков (внешние навыки)

### Маркетплейс ClawhHub

Установка навыков сообщества с [ClawhHub](https://clawhub.ai):

```
# Через дашборд: Skills → External → Search
# Через чат: установка по URL
```

API маркетплейса (`clawhub.ai/api/v1`) поддерживает:
- `Search(query)` — результаты, ранжированные по BM25-релевантности
- `Resolve(slug)` — получение URL загрузки и контрольной суммы
- `Download()` — загрузка архива (максимум 50 МБ)

### Процесс установки

1. Проверка лимита `MaxInstalled`
2. Разрешение из источника (ClawhHub, URL или локальная директория)
3. Валидация slug по `^[a-zA-Z0-9][a-zA-Z0-9_-]{0,63}$`
4. Загрузка и проверка контрольной суммы SHA-256
5. Разбор `SKILL.md` с расширенным frontmatter
6. Проверка уровня изоляции по конфигурации
7. Сканирование на паттерны prompt injection
8. Атомарное перемещение в директорию установки
9. Регистрация в реестре навыков

### Уровни изоляции

| Уровень | Поведение | Подтверждение |
|---------|----------|---------------|
| `text_only` | Только внедрение в системный промпт | Авто |
| `shell` | Выполнение через `ShellExecutor` | Ручное |
| `docker` | Выполнение в Docker-контейнере | Подтверждение |
| `wasm` | Рантайм WebAssembly | Авто |

**Цепочка резервов**: Если навык требует shell, но shell_exec отключён, используется `webfetchProxySkill` (извлекает URL из промпта и загружает их), затем `text_only`.

### Безопасность

- Валидация slug предотвращает обход пути
- Проверка контрольной суммы для удалённых загрузок
- Проверка уровня изоляции по конфигурации (`AllowShell`, `AllowDocker`, `AllowWASM`)
- Обнаружение файлов с кодом: отклоняет навыки с файлами `.py`/`.js`/`.go`/и т.д., если не изолированы должным образом
- Сканирование prompt injection: предупреждает о подозрительных паттернах в теле навыка

## Горячая перезагрузка навыков

Навыки поддерживают изменение конфигурации во время выполнения без перезапуска:

1. Навыки вызывают `RegisterKey()` при запуске для объявления своих ключей конфигурации
2. Редактор конфигурации дашборда вызывает `Store.Set()`, который публикует `ConfigChanged`
3. Шина событий передаёт в `registry.DispatchConfigChanged()`
4. Навыки, реализующие `ConfigReloadable`, получают новое значение

**Критическое правило**: навыки ДОЛЖНЫ регистрироваться безусловно (не внутри `if apiKey != ""`). Вместо этого используйте контроль через capability: `AddCapability("web")` при наличии API-ключа, `RemoveCapability("web")` при его удалении.

## Принудительные триггеры

Навыки могут объявлять ключевые слова для принудительного вызова:

```yaml
force_triggers: [weather, погода, météo]
```

Когда сообщение пользователя содержит триггерное ключевое слово (регистронезависимый поиск подстроки), `ForceTool` устанавливается в LLM-запросе для итерации 0. Это гарантирует, что LLM всегда вызовет инструмент, а не ответит из обучающих данных.

Триггеры памяти (напр., "remember", "запомни") настраиваются отдельно и принудительно вызывают инструмент `remember`.
