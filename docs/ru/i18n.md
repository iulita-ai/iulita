# Интернационализация (i18n) и локализация (l10n)

Iulita поддерживает 6 языков на всех уровнях: системные сообщения бэкенда, консольный TUI, вывод навыков, словарь подтверждений и фронтенд-дашборд Vue 3.

## Поддерживаемые языки

| Код | Язык | RTL | Письменность |
|-----|------|-----|-------------|
| `en` | English | Нет | Латиница |
| `ru` | Russian | Нет | Кириллица |
| `zh` | Chinese (Mandarin) | Нет | Ханьцзы |
| `es` | Spanish | Нет | Латиница |
| `fr` | French | Нет | Латиница |
| `he` | Hebrew | Да | Иврит |

## i18n на бэкенде

### Структура пакета

```
internal/i18n/
├── i18n.go        # Bundle, Init(), T(), Tl(), Translate()
├── context.go     # WithLocale(), LocaleFrom()
├── resolve.go     # ResolveLocale(), SupportedLanguages, matcher
└── catalog/
    ├── en.toml
    ├── ru.toml
    ├── zh.toml
    ├── es.toml
    ├── fr.toml
    └── he.toml
```

### Инициализация

`i18n.Init()` должен вызываться после миграций БД и перед ассистентом/навыками. Он:

1. Создаёт бандл `go-i18n/v2` с английским в качестве языка по умолчанию
2. Регистрирует TOML-анмаршалер
3. Читает все встроенные `*.toml` каталоги из `catalog/`
4. Предварительно кэширует экземпляры `*i18n.Localizer` для всех поддерживаемых языков
5. Разбирает слова подтверждения в таблицы O(1)-поиска

### API перевода

```go
// На основе контекста (читает локаль из контекста)
i18n.T(ctx, "ConsolePlaceholder")
i18n.T(ctx, "WeatherForTitle", map[string]interface{}{"Location": "Moscow"})

// Прямой по тегу (без контекста)
i18n.Tl(language.Russian, "LocaleSwitched", map[string]interface{}{"Language": "Russian"})

// Базовая функция перевода
i18n.Translate(tag, msgID, data...)
```

Цепочка резервов: перевод для конкретной локали -> перевод на английский -> сырой msgID.

### Распространение контекста

```go
// Установка локали в контексте
ctx = i18n.WithLocale(ctx, language.Russian)

// Чтение локали из контекста
tag := i18n.LocaleFrom(ctx) // возвращает language.English, если не установлено
```

Локаль проходит через весь жизненный цикл запроса:

```
user_channels.locale (БД)
    ↓
msg.Locale (поле IncomingMessage)
    ↓
i18n.ResolveLocale(msg.Locale, msg.LanguageCode)
    ↓
i18n.WithLocale(ctx, tag)
    ↓
Все последующие вызовы i18n.T(ctx, ...)
```

### Определение локали

`ResolveLocale(channelLocale, msgLang)`:

1. Разбор `channelLocale` (сохранённое в БД предпочтение, напр. `"ru"`)
2. Резервный `msgLang` (язык платформы, напр. `LanguageCode` Telegram)
3. Прогон через `language.NewMatcher(SupportedLanguages)` для наилучшего совпадения
4. **Приоритет**: локаль из БД > язык платформы > английский по умолчанию

### Ключи каталога

Все 6 локалей покрывают следующие группы ключей:

| Группа | Количество | Примеры |
|--------|-----------|---------|
| Процесс подтверждения | 9 | `ApprovalAffirmative`, `ApprovalNegative`, `ApprovalPrompt` |
| Telegram | 7 | `TelegramHistoryCleared`, `TelegramRateLimited` |
| Console TUI | 17 | `ConsolePlaceholder`, `ConsoleThinking`, `ConsoleProcessing` |
| Консольные команды | 12 | `CommandHelpHeader`, `CommandStatusSkills` |
| Погода | 30+ | Коды погоды WMO, `WeatherLocationPrompt` |
| Навык локали | 3 | `LocaleSwitched`, `LocaleUnknown` |
| Ассистент | 3 | `AssistantLanguageDirective`, `AssistantCompressionPrompt` |
| Закладки | 4 | `BookmarkButton`, `BookmarkSaved`, `BookmarkAlreadySaved`, `BookmarkError` |
| Ошибки | 1 | `ErrorUnknownSkill` |

### Слова подтверждения

Каждая локаль определяет утвердительные и отрицательные слова для процесса подтверждения:

```toml
# ru.toml
[ApprovalAffirmative]
other = "да,д,ок,подтвердить,подтверждаю,yes,y,ok,confirm"

[ApprovalNegative]
other = "нет,н,отмена,отменить,no,n,cancel"
```

**Кроссъязыковая гарантия**: Все неанглийские локали включают английские слова как резерв. Пользователи могут набирать "yes" независимо от настройки локали.

Проверка подтверждения выполняется за O(1) с использованием предварительно разобранных наборов слов (`map[string]map[string]bool`).

### Языковая директива LLM

Когда локаль не английская, языковая директива внедряется в динамический системный промпт:

```
# Русский:
"Отвечай на русском языке. Язык ответов должен соответствовать предпочтениям пользователя."

# Иврит:
"השב בעברית. שפת התשובות חייבת להתאים להעדפת המשתמש."
```

Директива отображается на целевом языке, чтобы LLM видел инструкции на ожидаемом языке ответа. Заголовки разделов (`## Current Time`, `## Remembered Facts`) остаются на английском — LLM понимает их в любом случае.

### Особенности TOML-каталогов

- **Китайские кавычки**: Используйте скобки `「」` вместо `""` в `zh.toml` — TOML-парсер не обрабатывает полноширинные кавычки внутри обычных строк
- **Промпт сжатия**: Остаётся на английском во всех локалях — LLM нормально интерпретирует инструкции суммаризации на английском
- **Префикс резюме**: Переведён (появляется непосредственно в чате)

## Хранение локали по каналам

Локаль хранится по каналам, а не по пользователям:

```sql
user_channels (
    ...
    locale TEXT NOT NULL DEFAULT 'en'
)
```

Это позволяет использовать разные языки на разных каналах (напр., русский в Telegram, английский в WebChat).

### Операции с хранилищем

- `UpdateChannelLocale(ctx, chatID, locale)` — вызывается навыком `set_language`
- `GetChannelLocale(ctx, channelType, channelUserID)` — вызывается при обработке сообщений
- `GetChannelLocaleByChatID(ctx, chatID)` — альтернативный поиск

### Навык set_language

Пользователи переключают язык через чат:

```
Пользователь: "switch to Russian"
Бот: "Язык интерфейса изменён на русский."  (на русском)

Пользователь: "переключи на иврит"
Бот: "שפת הממשק שונתה לעברית."  (на иврите)
```

Навык:
1. Разбирает языковой ввод (BCP-47 код или название на английском/русском)
2. Валидирует по списку поддерживаемых языков
3. Обновляет `user_channels.locale` в БД
4. Возвращает подтверждение на **новом** языке

## i18n на фронтенде

### Настройка

```typescript
// ui/src/i18n/index.ts
const i18n = createI18n({
    legacy: false,  // режим Composition API
    locale: localStorage.getItem('iulita.locale') || navigator.language.split('-')[0] || 'en',
    fallbackLocale: 'en',
    messages: { en, ru, zh, es, fr, he }
})
```

Все 6 JSON-файлов локалей статически импортируются и включаются в бандл при сборке.

### Composable useLocale

```typescript
// ui/src/composables/useLocale.ts
const { currentLocale, isRTL, direction, setLocale } = useLocale()
```

- `currentLocale` — реактивный ref, общий для всех экземпляров composable
- `isRTL` — computed: `true` для иврита
- `direction` — computed: `'rtl'` или `'ltr'`
- `setLocale(tag)`:
  1. Обновляет ref `currentLocale`
  2. Устанавливает `locale.value` (vue-i18n)
  3. Сохраняет в `localStorage`
  4. Устанавливает `document.documentElement.lang`
  5. По возможности `PATCH /api/auth/locale` для синхронизации с бэкендом

### Поддержка RTL

RTL применяется покомпонентно (не на `<html>`), чтобы не ломать раскладку боковой панели. Компоненты используют computed-значение `direction` для установки атрибутов `dir` где необходимо.

### Компонент LanguagePicker

Выпадающий список Naive UI `<NSelect>` в боковой панели с 6 вариантами на родных алфавитах (напр., "עברית" для иврита). При выборе вызывает `setLocale()`.

### Структура JSON-локалей

Вложенные объекты пространств имён:

```json
{
  "nav": { "dashboard": "Dashboard", "facts": "Facts", ... },
  "common": { "save": "Save", "cancel": "Cancel", ... },
  "login": { "title": "Login", ... },
  "dashboard": { ... },
  "tasks": { ... },
  "settings": { ... },
  "locale": { "en": "English", "ru": "Русский", "he": "עברית", ... }
}
```

### Синхронизация локали с бэкендом

`PATCH /api/auth/locale` (эндпоинт дашборда):
1. Валидирует локаль через `i18n.IsSupported()`
2. Обновляет `user_channels.locale` для всех каналов пользователя
3. Возвращает обновлённый профиль пользователя

## Добавление нового языка

### Бэкенд

1. Создайте `internal/i18n/catalog/<code>.toml` со всеми необходимыми ключами
2. Добавьте `language.Tag` в `SupportedLanguages` в `resolve.go`
3. Добавьте флаг RTL при необходимости
4. Добавьте соответствия названий языков в `skill/locale/locale.go` (английские + русские названия)

### Фронтенд

1. Создайте `ui/src/i18n/locales/<code>.json` со всеми пространствами имён
2. Импортируйте и зарегистрируйте в `ui/src/i18n/index.ts`
3. Добавьте в RTL_LOCALES в `ui/src/composables/useLocale.ts`, если RTL
4. Добавьте нативную метку во все файлы локалей в `locale.<code>`
