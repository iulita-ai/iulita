# 国际化（i18n）与本地化（l10n）

Iulita 在所有层面支持 6 种语言：后端系统消息、控制台 TUI、技能输出、审批词汇，以及 Vue 3 仪表盘前端。

## 支持的语言

| 代码 | 语言 | RTL | 文字 |
|------|------|-----|------|
| `en` | English | 否 | 拉丁字母 |
| `ru` | Russian | 否 | 西里尔字母 |
| `zh` | Chinese（普通话） | 否 | 汉字 |
| `es` | Spanish | 否 | 拉丁字母 |
| `fr` | French | 否 | 拉丁字母 |
| `he` | Hebrew | 是 | 希伯来字母 |

## 架构

```
后端（Go）                             前端（Vue 3）
┌─────────────────────┐              ┌─────────────────────┐
│ i18n 包              │              │ vue-i18n v9         │
│ ├── Bundle          │              │ ├── JSON 语言文件    │
│ ├── TOML 目录        │              │ ├── useLocale()     │
│ └── 上下文区域设置    │              │ └── LanguagePicker  │
└─────────────────────┘              └─────────────────────┘
         ↕                                    ↕
   user_channels.locale              localStorage + PATCH /api/auth/locale
```

## 后端 i18n

### 包结构

```
internal/i18n/
├── i18n.go        # Bundle、Init()、T()、Tl()、Translate()
├── context.go     # WithLocale()、LocaleFrom()
├── resolve.go     # ResolveLocale()、SupportedLanguages、matcher
└── catalog/
    ├── en.toml
    ├── ru.toml
    ├── zh.toml
    ├── es.toml
    ├── fr.toml
    └── he.toml
```

### 初始化

`i18n.Init()` 必须在数据库迁移之后、助手/技能之前调用。它：

1. 创建以英语为默认语言的 `go-i18n/v2` bundle
2. 注册 TOML 解析器
3. 从 `catalog/` 读取所有嵌入的 `*.toml` 目录文件
4. 为所有支持的语言预缓存 `*i18n.Localizer` 实例
5. 将审批词汇解析为 O(1) 查找表

### 翻译 API

```go
// 基于上下文（从上下文读取区域设置）
i18n.T(ctx, "ConsolePlaceholder")
i18n.T(ctx, "WeatherForTitle", map[string]interface{}{"Location": "Moscow"})

// 直接基于标签（无上下文）
i18n.Tl(language.Russian, "LocaleSwitched", map[string]interface{}{"Language": "Russian"})

// 核心翻译函数
i18n.Translate(tag, msgID, data...)
```

回退链：区域特定翻译 → 英语翻译 → 原始 msgID。

### 上下文传播

```go
// 在上下文中设置区域设置
ctx = i18n.WithLocale(ctx, language.Russian)

// 从上下文读取区域设置
tag := i18n.LocaleFrom(ctx) // 未设置时返回 language.English
```

区域设置贯穿整个请求生命周期：

```
user_channels.locale（数据库）
    ↓
msg.Locale（IncomingMessage 字段）
    ↓
i18n.ResolveLocale(msg.Locale, msg.LanguageCode)
    ↓
i18n.WithLocale(ctx, tag)
    ↓
所有下游的 i18n.T(ctx, ...) 调用
```

### 区域设置解析

`ResolveLocale(channelLocale, msgLang)`：

1. 解析 `channelLocale`（数据库存储的偏好，如 `"ru"`）
2. 回退到 `msgLang`（平台语言，如 Telegram 的 `LanguageCode`）
3. 通过 `language.NewMatcher(SupportedLanguages)` 进行最佳匹配
4. **优先级**：数据库存储的区域设置 > 平台语言 > 英语默认值

### 目录键

所有 6 种语言覆盖以下键组：

| 组 | 数量 | 示例 |
|------|------|------|
| 审批流程 | 9 | `ApprovalAffirmative`、`ApprovalNegative`、`ApprovalPrompt` |
| Telegram | 7 | `TelegramHistoryCleared`、`TelegramRateLimited` |
| 控制台 TUI | 17 | `ConsolePlaceholder`、`ConsoleThinking`、`ConsoleProcessing` |
| 控制台命令 | 12 | `CommandHelpHeader`、`CommandStatusSkills` |
| 天气 | 30+ | WMO 天气代码、`WeatherLocationPrompt` |
| 区域设置技能 | 3 | `LocaleSwitched`、`LocaleUnknown` |
| 助手 | 3 | `AssistantLanguageDirective`、`AssistantCompressionPrompt` |
| 书签 | 4 | `BookmarkButton`、`BookmarkSaved`、`BookmarkAlreadySaved`、`BookmarkError` |
| 错误 | 1 | `ErrorUnknownSkill` |

### 审批词汇

每种语言为审批流程定义肯定和否定词汇：

```toml
# ru.toml
[ApprovalAffirmative]
other = "да,д,ок,подтвердить,подтверждаю,yes,y,ok,confirm"

[ApprovalNegative]
other = "нет,н,отмена,отменить,no,n,cancel"
```

**跨语言保证**：所有非英语语言都包含英语词汇作为回退。用户可以在任何区域设置下输入 "yes"。

审批检查使用预解析的词集（`map[string]map[string]bool`），时间复杂度为 O(1)。

### LLM 语言指令

当区域设置非英语时，语言指令会注入到动态系统提示词中：

```
# 俄语：
"Отвечай на русском языке. Язык ответов должен соответствовать предпочтениям пользователя."

# 希伯来语：
"השב בעברית. שפת התשובות חייבת להתאים להעדפת המשתמש."
```

指令以目标语言渲染，使 LLM 看到与预期响应语言匹配的指令。章节标题（`## Current Time`、`## Remembered Facts`）保持英语 — LLM 无论如何都能理解。

### TOML 目录注意事项

- **中文引号**：在 `zh.toml` 中使用 `「」` 括号代替 `""` — TOML 解析器无法处理常规字符串中的全角引号
- **压缩提示词**：所有语言保持英语 — LLM 可以正确解释英语摘要指令
- **摘要前缀**：已翻译（直接出现在聊天中）

## 每通道区域设置存储

区域设置按通道存储，而非按用户：

```sql
user_channels (
    ...
    locale TEXT NOT NULL DEFAULT 'en'
)
```

这允许在不同通道上使用不同语言（如 Telegram 上用俄语，WebChat 上用英语）。

### 存储操作

- `UpdateChannelLocale(ctx, chatID, locale)` — 由 `set_language` 技能调用
- `GetChannelLocale(ctx, channelType, channelUserID)` — 在消息处理期间调用
- `GetChannelLocaleByChatID(ctx, chatID)` — 替代查找方式

### set_language 技能

用户通过聊天切换语言：

```
用户："switch to Russian"
机器人："Язык интерфейса изменён на русский."（俄语）

用户："переключи на иврит"
机器人："שפת הממשק שונתה לעברית."（希伯来语）
```

该技能：
1. 解析语言输入（BCP-47 代码或英语/俄语名称）
2. 根据支持的语言验证
3. 更新数据库中的 `user_channels.locale`
4. 以**新**语言返回确认

## 前端 i18n

### 设置

```typescript
// ui/src/i18n/index.ts
const i18n = createI18n({
    legacy: false,  // Composition API 模式
    locale: localStorage.getItem('iulita.locale') || navigator.language.split('-')[0] || 'en',
    fallbackLocale: 'en',
    messages: { en, ru, zh, es, fr, he }
})
```

所有 6 个 JSON 语言文件在构建时静态导入并打包。

### useLocale 组合式函数

```typescript
// ui/src/composables/useLocale.ts
const { currentLocale, isRTL, direction, setLocale } = useLocale()
```

- `currentLocale` — 响应式 ref，跨所有组合式函数实例共享
- `isRTL` — 计算属性：Hebrew 为 `true`
- `direction` — 计算属性：`'rtl'` 或 `'ltr'`
- `setLocale(tag)`：
  1. 更新 `currentLocale` ref
  2. 设置 `locale.value`（vue-i18n）
  3. 保存到 `localStorage`
  4. 设置 `document.documentElement.lang`
  5. 尽力调用 `PATCH /api/auth/locale` 同步后端

### RTL 支持

RTL 按组件应用（而非 `<html>`）以避免破坏侧边栏布局。组件使用 `direction` 计算值在需要时设置 `dir` 属性。

### LanguagePicker 组件

侧边栏中的 Naive UI `<NSelect>` 下拉菜单，提供 6 个以原生文字标注的选项（如 Hebrew 为 "עברית"）。选择时调用 `setLocale()`。

### JSON 语言文件结构

嵌套命名空间对象：

```json
{
  "nav": { "dashboard": "Dashboard", "facts": "Facts", ... },
  "common": { "save": "Save", "cancel": "Cancel", ... },
  "login": { "title": "Login", ... },
  "dashboard": { ... },
  "tasks": { ... },
  "settings": { ... },
  "locale": { "en": "English", "ru": "Русский", "he": "עברית", ... }
}
```

### 后端区域设置同步

`PATCH /api/auth/locale`（仪表盘端点）：
1. 通过 `i18n.IsSupported()` 验证区域设置
2. 更新用户所有通道的 `user_channels.locale`
3. 返回更新后的用户配置文件

## 添加新语言

### 后端

1. 创建 `internal/i18n/catalog/<code>.toml`，包含所有必需的键
2. 在 `resolve.go` 中将 `language.Tag` 添加到 `SupportedLanguages`
3. 如需要添加 RTL 标志
4. 在 `skill/locale/locale.go` 中添加语言名称映射（英语 + 俄语名称）

### 前端

1. 创建 `ui/src/i18n/locales/<code>.json`，包含所有命名空间
2. 在 `ui/src/i18n/index.ts` 中导入并注册
3. 如果是 RTL，添加到 `ui/src/composables/useLocale.ts` 的 RTL_LOCALES 中
4. 在所有语言文件的 `locale.<code>` 下添加原生标签
