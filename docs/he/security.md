# אבטחה

## אימות

### JWT

- **אלגוריתם**: HMAC-SHA256
- **TTL טוקן גישה**: 24 שעות
- **TTL טוקן רענון**: 7 ימים
- **Claims**: `user_id`, `username`, `role`
- **סוד**: 32 בתים hex שנוצרו אוטומטית אם לא מוגדר
- **אחסון**: keyring (macOS Keychain / Linux SecretService) או משתנה סביבה `IULITA_JWT_SECRET`

### סיסמאות

- **גיבוב**: bcrypt עם עלות ברירת מחדל
- **אתחול**: משתמש ראשון מקבל סיסמה אקראית עם `MustChangePass: true`
- **לוח בקרה**: endpoint לשינוי סיסמה ב-`POST /api/auth/change-password`

### Middleware

1. `FiberMiddleware` — מאמת `Authorization: Bearer <token>`, שומר claims ב-fiber locals
2. `AdminOnly` — בודק `role == admin`, מחזיר 403 אם לא

## ניהול סודות

### שכבות אחסון

| סוד | משתנה סביבה | Keyring | קובץ חלופי |
|--------|-------------|---------|---------------|
| מפתח API של Claude | `IULITA_CLAUDE_API_KEY` | `claude-api-key` | — |
| טוקן Telegram | `IULITA_TELEGRAM_TOKEN` | `telegram-token` | — |
| סוד JWT | `IULITA_JWT_SECRET` | `jwt-secret` | יצירה אוטומטית |
| מפתח הצפנת הגדרות | `IULITA_CONFIG_KEY` | `config-encryption-key` | קובץ `encryption.key` |

**סדר פתרון**: משתנה סביבה → keyring → קובץ חלופי → יצירה אוטומטית.

### הצפנת הגדרות (AES-256-GCM)

דריסות הגדרות בזמן ריצה במסד הנתונים ניתנות להצפנה:

- **אלגוריתם**: AES-256-GCM (הצפנה מאומתת)
- **Nonce**: 12 בתים, נוצר באקראי לכל הצפנה
- **פורמט**: `base64(nonce ‖ ciphertext)`
- **הצפנה אוטומטית**: מפתחות שהוכרזו כ-`secret_keys` במניפסטים של SKILL.md
- **בטיחות API**: לוח הבקרה לעולם לא מחזיר ערכים מפוענחים למפתחות סודיים
- **דחיית מצייני מקום**: ערכי `"***"` או ריקים נדחים למפתחות סודיים

## הגנת SSRF

כל בקשות HTTP יוצאות (שליפת אינטרנט, חיפוש אינטרנט, מיומנויות חיצוניות) עוברות הגנת SSRF.

### טווחי IP חסומים

| טווח | סוג |
|-------|------|
| `10.0.0.0/8` | RFC1918 פרטי |
| `172.16.0.0/12` | RFC1918 פרטי |
| `192.168.0.0/16` | RFC1918 פרטי |
| `100.64.0.0/10` | Carrier-grade NAT (RFC 6598) |
| `fc00::/7` | IPv6 Unique Local |
| `127.0.0.0/8` | Loopback |
| `::1/128` | IPv6 loopback |
| `169.254.0.0/16` | Link-local |
| `fe80::/10` | IPv6 link-local |
| טווחי Multicast | הכל |

כתובות IPv6 ממופות IPv4 מנורמלות ל-IPv4 לפני הבדיקה.

### הגנה דו-שכבתית (ללא פרוקסי)

**שכבה 1 — DNS מוקדם**: לפני חיבור, כל כתובות ה-IP עבור hostname מפורשות. אם כתובת כלשהי פרטית, החיבור נדחה.

**שכבה 2 — בקרה בזמן חיבור**: פונקציית `net.Dialer.Control` בודקת את ה-IP בפועל שנפתר בזמן החיבור. זה תופס **התקפות DNS rebinding** שבהן hostname מפנה ל-IP ציבורי בבדיקה מוקדמת אך מפנה מחדש ל-IP פרטי לפני החיבור בפועל.

### נתיב פרוקסי

כאשר פרוקסי מוגדר (`proxy.url`), גישת ה-dialer לא ניתנת לשימוש (לפרוקסי עצמו עשוי להיות IP פרטי באשכולות Kubernetes). במקום זאת:

- `ssrfTransport.RoundTrip` מבצע בדיקת URL מוקדמת בלבד
- חיבור הפרוקסי ל-IP פרטי מותר (מכוון לפרוקסים פנימיים באשכול)
- כתובות URL יעד ל-IP פרטי עדיין חסומות

### זיהוי פרוקסי אקטיבי

`isProxyActive()` קורא בפועל לפונקציית הפרוקסי עם בקשת מבחן (לא רק `Proxy != nil`), מכיוון ש-`http.DefaultTransport` תמיד מגדיר `Proxy = ProxyFromEnvironment`.

## רמות אישור כלים

| רמה | התנהגות | מיומנויות |
|-------|----------|--------|
| `ApprovalAuto` | ביצוע מיידי | רוב המיומנויות (ברירת מחדל) |
| `ApprovalPrompt` | המשתמש חייב לאשר | מבצע Docker |
| `ApprovalManual` | מנהל חייב לאשר | ביצוע shell |

### זרימה

1. מיומנות מצהירה על רמתה דרך ממשק `ApprovalDeclarer`
2. לפני ביצוע, העוזר בודק `registry.ApprovalLevelFor(toolName)`
3. עבור `Prompt`/`Manual`: שומר קריאת כלי ממתינה ב-`approvalStore`
4. שולח הנחיית אישור למשתמש
5. מחזיר `"awaiting approval"` ל-LLM (לא חוסם)
6. ההודעה הבאה של המשתמש נבדקת מול אוצר מילים לאישור מותאם לשפה
7. אם אושר: מבצע את קריאת הכלי השמורה, מחזיר תוצאה
8. אם נדחה: מחזיר "cancelled"

### אוצר מילים מותאם לשפה

מילות אישור מוגדרות בכל 6 קטלוגי השפה וכוללות אנגלית כחלופה:

```
# רוסית חיובי:
да, д, ок, подтвердить, подтверждаю, yes, y, ok, confirm

# עברית שלילי:
לא, ביטול, בטל, no, n, cancel
```

## אבטחת Telegram

- **רשימה לבנה של משתמשים**: `telegram.allowed_ids` מגביל מי יכול לשוחח עם הבוט
- **רשימה לבנה ריקה**: מאפשר לכל המשתמשים (אזהרה נרשמת)
- **הגבלת קצב**: מגביל קצב חלון הזזה לכל שיחה

## אבטחת Shell Exec

למיומנות `shell_exec` האבטחה הקפדנית ביותר:

- **רמת אישור**: `ApprovalManual` (נדרש אישור מנהל)
- **רשימה לבנה בלבד**: רק קבצים הרצה ב-`AllowedBins` יכולים לרוץ
- **נתיבים אסורים**: רשימה ניתנת להגדרה של נתיבים שלא יכולים להופיע בארגומנטים
- **חצייה בנתיבים**: `..` בארגומנטים נדחה
- **מגבלת פלט**: מקסימום 16KB
- **תיקיית עבודה**: `os.TempDir()` (לא תיקיית הפרויקט)

## הגבלת קצב

### מגביל קצב לכל שיחה

חלון הזזה: עוקב אחר חותמות זמן לכל `chatID`. אם מספר ההודעות בתוך `window` חורג מ-`rate`, ההודעה נדחית.

### מגביל פעולות גלובלי

חלון קבוע: סך כל פעולות LLM/כלים לשעה בכל השיחות. מתאפס אוטומטית בגבול החלון.

## מעקב עלויות

- **בזיכרון**: עלות יומית עם mutex, מתאפס אוטומטית בגבול היום
- **קבוע**: `IncrementUsageWithCost` שומר לטבלת `usage_stats`
- **מגבלה יומית**: `cost.daily_limit_usd` (0 = ללא הגבלה)
- **תמחור לכל מודל**: `config.ModelPrice{InputPerMillion, OutputPerMillion}`

## אבטחת CI/CD

- **pre-commit hook**: חוסם סודות דרך [gitleaks](https://github.com/gitleaks/gitleaks)
- **CI**: פעולת gitleaks סורקת את כל ה-commits
- **CodeQL**: שאילתות security-extended עבור Go ו-JavaScript/TypeScript (כשהמאגר ציבורי)
- **תלויות**: התראות Dependabot (הפעל בהגדרות GitHub)

## אבטחת מיומנויות חיצוניות

- **אימות slug**: `^[a-zA-Z0-9][a-zA-Z0-9_-]{0,63}$` — מונע חצייה בנתיבים
- **אימות checksum**: SHA-256 להורדות מרוחקות
- **אימות בידוד**: מיומנויות חייבות להצהיר על רמת בידוד, נבדק מול דגלי הגדרות
- **זיהוי קוד**: דוחה מיומנויות עם קבצי קוד אלא אם מבודדים כהלכה
- **סריקת הזרקת פרומפט**: מזהיר על דפוסים חשודים בגוף המיומנות
- **גודל ארכיון מקסימלי**: ניתן להגדרה (ברירת מחדל 50MB עבור ClawhHub)
