# תזמור רב-סוכנים

Iulita תומכת בהרצה מקבילית של תת-סוכנים לפירוק משימות מורכבות. ה-LLM מחליט באופן אוטונומי מתי להשתמש בתזמור בהתבסס על מורכבות המשימה.

## סקירה

המיומנות `orchestrate` משיקה מספר תת-סוכנים מתמחים במקביל. כל תת-סוכן מריץ לולאה סוכנית מפושטת עם פרומפט מערכת משלו, תת-קבוצת כלים וניתוב אופציונלי של ספק LLM. התוצאות נאספות ומוחזרות לעוזר הראשי כדו"ח markdown מובנה.

## ארכיטקטורה

```
הודעת משתמש
    │
    ▼
עוזר (לולאה סוכנית ראשית)
    │ מחליט שנדרש תזמור
    │ קורא לכלי orchestrate עם מפרטי סוכנים
    ▼
מיומנות Orchestrate
    │ מאמתת עומק (מקסימום 1)
    │ בונה Budget מהגדרות + קלט
    ▼
מתזמר (הרצה מקבילית דרך errgroup)
    ├── Runner (agent_1: researcher) ──→ LLM ←→ כלים
    ├── Runner (agent_2: analyst)    ──→ LLM ←→ כלים
    └── Runner (agent_3: planner)    ──→ LLM ←→ כלים
         │
         │ תקציב טוקנים אטומי משותף
         │ timeout + הקשר לכל סוכן
         │ אירועי סטטוס → ערוצים
         ▼
    AgentResults שנאספו
    │ מעוצבים כ-markdown
    ▼
העוזר ממשיך עם פלט התזמור
```

## סוגי סוכנים

| סוג | מוקד הפרומפט | כלים ברירת מחדל | Route Hint |
|------|-------------|----------------|------------|
| `researcher` | איסוף מידע, חיפוש ברשת | `web_search`, `webfetch` | — |
| `analyst` | זיהוי דפוסים, חריגות, תובנות | הכל | — |
| `planner` | פירוק מטרות לשלבים מסודרים | `datetime` | — |
| `coder` | כתיבה, סקירה, דיבאג קוד | הכל | — |
| `summarizer` | דחיסת קלט לנקודות מהותיות | הכל | `ollama` |
| `generic` | כללי | הכל | — |

## מערכת תקציב

כל הסוכנים חולקים מונה `atomic.Int64` יחיד. התקרה היא **רכה** בעיצוב.

| פרמטר | ברירת מחדל | דריסה |
|--------|-----------|--------|
| מקסימום סיבובים | 10 | `Budget.MaxTurns` |
| timeout | 60 שניות | `Budget.Timeout` או קלט `timeout` |
| סוכנים מקבילים מקסימום | 5 | `Budget.MaxAgents` או הגדרות |
| תקציב טוקנים משותף | ללא הגבלה | `Budget.MaxTokens` או קלט `max_tokens` |

## אכיפת עומק

תת-סוכנים לא יכולים ליצור תת-סוכנים נוספים (`MaxDepth = 1`). נאכף על ידי בדיקת הקשר וסינון כלי `orchestrate`.

## אבטחה

- מיומנויות ApprovalManual/ApprovalPrompt מסוננות מתת-סוכנים
- רשימות לבנות אופציונליות של כלים לכל סוכן
- כלי `orchestrate` תמיד מוחרג מתת-סוכנים

## פרוטוקול אירועי סטטוס

| אירוע | מתי | שדות |
|-------|------|-------|
| `orchestration_started` | לפני השקת סוכנים | `agent_count` |
| `agent_started` | לכל סוכן, לפני הרצה | `agent_id`, `agent_type` |
| `agent_progress` | לכל סוכן, אחרי כל סיבוב LLM | `agent_id`, `turn` |
| `agent_completed` | לכל סוכן, בהצלחה | `agent_id`, `tokens`, `duration_ms` |
| `agent_failed` | לכל סוכן, בשגיאה | `agent_id`, `error` |
| `orchestration_done` | אחרי סיום כל הסוכנים | `success_count`, `total_tokens`, `duration_ms` |

אירועים לאחר סיום סוכנים משתמשים ב-`context.Background()` עם timeout של 5 שניות להבטחת משלוח גם אם מועד היעד של ההקשר האב פג.

## הגדרות

| מפתח | ברירת מחדל | תיאור |
|-------|-----------|--------|
| `skills.orchestrate.enabled` | true | הפעלה/השבתה |
| `skills.orchestrate.max_tokens` | 0 (ללא הגבלה) | תקרת תקציב טוקנים |
| `skills.orchestrate.max_agents` | 5 | סוכנים מקבילים מקסימום |
| `skills.orchestrate.timeout` | 60s | timeout לכל סוכן |
| `skills.orchestrate.request_timeout` | 1h | מועד יעד כולל לתזמור (מקסימום 4h) |

כל הערכים תומכים בטעינה חמה דרך `ConfigReloadable`.

## ממשק קדמי

רכיב `AgentProgress.vue` מציג סטטוס סוכנים בזמן אמת בתצוגת הצ'אט, עם אייקונים לפי סוג, שם, סטטוס והתקדמות.
