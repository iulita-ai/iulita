# מיומנויות

מיומנויות הן כלים שהעוזר יכול להפעיל במהלך שיחות. כל מיומנות חושפת כלי אחד או יותר ל-LLM עם שם, תיאור וסכמת קלט JSON.

## ממשק מיומנות

```go
type Skill interface {
    Name() string
    Description() string
    InputSchema() json.RawMessage  // nil for text-only skills
    Execute(ctx context.Context, input json.RawMessage) (string, error)
}
```

**ממשקים אופציונליים:**
- `CapabilityAware` — `RequiredCapabilities() []string`: המיומנות מוחרגת אם יכולת כלשהי חסרה
- `ConfigReloadable` — `OnConfigChanged(key, value string)`: נקרא בשינוי הגדרות בזמן ריצה
- `ApprovalDeclarer` — `ApprovalLevel() ApprovalLevel`: דרישת אישור

## רמות אישור

| רמה | התנהגות | בשימוש על ידי |
|-------|----------|---------|
| `ApprovalAuto` | ביצוע מיידי (ברירת מחדל) | רוב המיומנויות |
| `ApprovalPrompt` | המשתמש חייב לאשר בצ'אט | מבצע Docker |
| `ApprovalManual` | מנהל חייב לאשר | ביצוע shell |

זרימת האישור היא **לא חוסמת**: המיומנות מחזירה "ממתין לאישור" ל-LLM. ההודעה הבאה של המשתמש נבדקת מול אוצר מילים לאישור מותאם לשפה (כן/לא ב-6 שפות).

## מיומנויות טקסט (הזרקת פרומפט מערכת)

מיומנויות יכולות להיות אך ורק הזרקת פרומפט מערכת — ללא שיטת `Execute`, ללא הגדרת כלי ל-LLM.

### פורמט SKILL.md

```yaml
---
name: my-skill
description: What this skill does
capabilities: [optional-cap]
config_keys: [skills.my-skill.setting]
secret_keys: [skills.my-skill.api_key]
force_triggers: [keyword1, keyword2]
---

Markdown instructions injected into the system prompt.
```

- מיומנויות עם `InputSchema() == nil` תורמות רק ל-`staticSystemPrompt()`
- גוף ה-Markdown הופך לחלק מהוראות ה-LLM
- `force_triggers` כופים קריאות כלים ספציפיות כאשר מילות מפתח תואמות את הודעת המשתמש

### נתיבי טעינה

1. **מוטבע בחבילות Go**: `//go:embed SKILL.md` + `LoadManifestFromFS()`
2. **תיקייה חיצונית**: `LoadExternalManifests(dir)` מ-`~/.local/share/iulita/skills/`
3. **חיצוני מותקן**: דרך שוק ClawhHub או URL

## מנהל מיומנויות (מיומנויות חיצוניות)

### שוק ClawhHub

התקן מיומנויות קהילתיות מ-[ClawhHub](https://clawhub.ai):

```
# דרך לוח הבקרה: Skills → External → Search
# דרך צ'אט: התקן מ-URL
```

API השוק (`clawhub.ai/api/v1`) תומך ב:
- `Search(query)` — תוצאות מדורגות לפי רלוונטיות BM25
- `Resolve(slug)` — קבלת URL הורדה ו-checksum
- `Download()` — הורדת ארכיון (מקסימום 50MB)

### זרימת התקנה

1. בדיקת מגבלת `MaxInstalled`
2. פתרון ממקור (ClawhHub, URL או תיקייה מקומית)
3. אימות slug מול `^[a-zA-Z0-9][a-zA-Z0-9_-]{0,63}$`
4. הורדה ואימות checksum SHA-256
5. ניתוח `SKILL.md` עם frontmatter מורחב
6. אימות רמת בידוד מול הגדרות
7. סריקה לדפוסי הזרקת פרומפט
8. העברה אטומית לתיקיית התקנה
9. רישום ב-skill registry

### רמות בידוד

| רמה | התנהגות | אישור |
|-------|----------|----------|
| `text_only` | הזרקת פרומפט מערכת בלבד | אוטומטי |
| `shell` | ביצוע shell דרך `ShellExecutor` | ידני |
| `docker` | ביצוע בקונטיינר Docker | הנחיה |
| `wasm` | runtime WebAssembly | אוטומטי |

**שרשרת חלופות**: אם מיומנות דורשת shell אך shell_exec מושבת, היא נופלת ל-`webfetchProxySkill` (חילוץ URL מהפרומפט ושליפתם), ואז ל-`text_only`.

### אבטחה

- אימות slug מונע חצייה בנתיבים
- אימות checksum להורדות מרוחקות
- אימות רמת בידוד מול הגדרות (`AllowShell`, `AllowDocker`, `AllowWASM`)
- זיהוי קבצי קוד: דוחה מיומנויות עם קבצי `.py`/`.js`/`.go` וכו' אלא אם מבודדים כהלכה
- סריקת הזרקת פרומפט: מזהיר על דפוסים חשודים בגוף המיומנות

## טעינה חמה של מיומנויות

מיומנויות תומכות בשינויי הגדרות בזמן ריצה ללא הפעלה מחדש:

1. מיומנויות קוראות `RegisterKey()` בהפעלה כדי להצהיר על מפתחות ההגדרה שלהן
2. עורך ההגדרות בלוח הבקרה קורא `Store.Set()` שמפרסם `ConfigChanged`
3. אפיק האירועים מעביר ל-`registry.DispatchConfigChanged()`
4. מיומנויות המממשות `ConfigReloadable` מקבלות את הערך החדש

**כלל קריטי**: מיומנויות חייבות להירשם ללא תנאי (לא בתוך `if apiKey != ""`). השתמש בהגבלת יכולות במקום: `AddCapability("web")` כשמפתח API קיים, `RemoveCapability("web")` כשמוסר.

## טריגרים כפויים

מיומנויות יכולות להצהיר על מילות מפתח שכופות הפעלת כלי:

```yaml
force_triggers: [weather, погода, météo]
```

כאשר הודעת המשתמש מכילה מילת מפתח טריגר (התאמת תת-מחרוזת ללא רגישות לרישיות), `ForceTool` מוגדר על בקשת ה-LLM לאיטרציה 0. זה מבטיח שה-LLM תמיד קורא לכלי במקום לענות מנתוני אימון.

טריגרי זיכרון (למשל "remember", "запомни") מוגדרים בנפרד וכופים את הכלי `remember`.
