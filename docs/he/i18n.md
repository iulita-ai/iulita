# בינלאומיות (i18n) ולוקליזציה (l10n)

Iulita תומך ב-6 שפות בכל השכבות: הודעות מערכת בצד השרת, TUI קונסולה, פלטי מיומנויות, אוצר מילים לאישור ולוח בקרה Vue 3 בצד הלקוח.

## שפות נתמכות

| קוד | שפה | RTL | כתב |
|------|----------|-----|--------|
| `en` | אנגלית | לא | לטיני |
| `ru` | רוסית | לא | קירילי |
| `zh` | סינית (מנדרינית) | לא | האנזי |
| `es` | ספרדית | לא | לטיני |

## i18n בצד השרת

### מבנה החבילה

```
internal/i18n/
├── i18n.go        # Bundle, Init(), T(), Tl(), Translate()
├── context.go     # WithLocale(), LocaleFrom()
├── resolve.go     # ResolveLocale(), SupportedLanguages, matcher
└── catalog/
    ├── en.toml
    ├── ru.toml
    ├── zh.toml
    ├── es.toml
    ├── fr.toml
    └── he.toml
```

### אתחול

`i18n.Init()` חייב להיקרא אחרי מיגרציות DB ולפני העוזר/מיומנויות. הוא:

1. יוצר bundle `go-i18n/v2` עם אנגלית כשפת ברירת מחדל
2. רושם את ה-unmarshaler של TOML
3. קורא את כל קטלוגי `*.toml` המוטבעים מ-`catalog/`
4. מאחסן מראש מופעי `*i18n.Localizer` לכל השפות הנתמכות
5. מפרש מילות אישור לטבלאות חיפוש O(1)

### API תרגום

```go
// מבוסס הקשר (קורא locale מ-context)
i18n.T(ctx, "ConsolePlaceholder")
i18n.T(ctx, "WeatherForTitle", map[string]interface{}{"Location": "Moscow"})

// מבוסס tag ישיר (ללא context)
i18n.Tl(language.Russian, "LocaleSwitched", map[string]interface{}{"Language": "Russian"})

// פונקציית תרגום ליבה
i18n.Translate(tag, msgID, data...)
```

שרשרת חלופות: תרגום ספציפי ל-locale → תרגום לאנגלית → msgID גולמי.

### הפצת הקשר

```go
// הגדרת locale בהקשר
ctx = i18n.WithLocale(ctx, language.Russian)

// קריאת locale מההקשר
tag := i18n.LocaleFrom(ctx) // מחזיר language.English אם לא מוגדר
```

ה-locale זורם דרך כל מחזור חיי הבקשה:

```
user_channels.locale (DB)
    ↓
msg.Locale (שדה IncomingMessage)
    ↓
i18n.ResolveLocale(msg.Locale, msg.LanguageCode)
    ↓
i18n.WithLocale(ctx, tag)
    ↓
כל קריאות i18n.T(ctx, ...) במורד הזרם
```

### פתרון Locale

`ResolveLocale(channelLocale, msgLang)`:

1. ניתוח `channelLocale` (העדפה שמורה ב-DB, למשל `"ru"`)
2. נפילה ל-`msgLang` (שפת פלטפורמה, למשל `LanguageCode` של Telegram)
3. הרצה דרך `language.NewMatcher(SupportedLanguages)` להתאמה מיטבית
4. **עדיפות**: locale מאוחסן ב-DB > שפת פלטפורמה > ברירת מחדל אנגלית

### מילות אישור

כל locale מגדיר מילות חיוב ושלילה עבור זרימת האישור:

```toml
# ru.toml
[ApprovalAffirmative]
other = "да,д,ок,подтвердить,подтверждаю,yes,y,ok,confirm"

[ApprovalNegative]
other = "нет,н,отмена,отменить,no,n,cancel"
```

**ערבות חוצת שפות**: כל ה-locales שאינם אנגלית כוללים מילות אנגלית כחלופה. משתמשים יכולים להקליד "yes" ללא קשר להגדרת ה-locale שלהם.

בדיקת האישור היא O(1) באמצעות סטי מילים מנותחים מראש (`map[string]map[string]bool`).

### הנחיית שפה ל-LLM

כאשר ה-locale אינו אנגלית, הנחיית שפה מוזרקת לפרומפט המערכת הדינמי:

```
# רוסית:
"Отвечай на русском языке. Язык ответов должен соответствовать предпочтениям пользователя."

# עברית:
"השב בעברית. שפת התשובות חייבת להתאים להעדפת המשתמש."
```

ההנחיה מעובדת בשפת היעד כך שה-LLM רואה הוראות התואמות לשפת התשובה הצפויה. כותרות סעיפים (`## Current Time`, `## Remembered Facts`) נשארות באנגלית — ה-LLM מבין אותן ללא קשר.

### הערות לקטלוג TOML

- **גרשיים סיניים**: השתמש בסוגריים `「」` במקום `""` ב-`zh.toml` — מפרש ה-TOML לא יכול לטפל בגרשיים ברוחב מלא בתוך מחרוזות רגילות
- **פרומפט דחיסה**: נשמר באנגלית בכל ה-locales — ה-LLM מפרש הוראות סיכום באנגלית היטב
- **קידומת סיכום**: מתורגמת (מופיעה ישירות בצ'אט)

## אחסון locale לכל ערוץ

Locale נשמר לכל ערוץ, לא לכל משתמש:

```sql
user_channels (
    ...
    locale TEXT NOT NULL DEFAULT 'en'
)
```

זה מאפשר שפות שונות בערוצים שונים (למשל רוסית ב-Telegram, אנגלית ב-WebChat).

### פעולות אחסון

- `UpdateChannelLocale(ctx, chatID, locale)` — נקרא על ידי מיומנות `set_language`
- `GetChannelLocale(ctx, channelType, channelUserID)` — נקרא בזמן עיבוד הודעות
- `GetChannelLocaleByChatID(ctx, chatID)` — חיפוש חלופי

### מיומנות set_language

משתמשים מחליפים שפה דרך צ'אט:

```
User: "switch to Russian"
Bot: "Язык интерфейса изменён на русский."  (ברוסית)

User: "переключи на иврит"
Bot: "שפת הממשק שונתה לעברית."  (בעברית)
```

המיומנות:
1. מפרשת קלט שפה (קוד BCP-47 או שם באנגלית/רוסית)
2. מאמתת מול שפות נתמכות
3. מעדכנת `user_channels.locale` ב-DB
4. מחזירה אישור בשפה **החדשה**

## i18n בצד הלקוח

### הגדרה

```typescript
// ui/src/i18n/index.ts
const i18n = createI18n({
    legacy: false,  // Composition API mode
    locale: localStorage.getItem('iulita.locale') || navigator.language.split('-')[0] || 'en',
    fallbackLocale: 'en',
    messages: { en, ru, zh, es, fr, he }
})
```

כל 6 קבצי locale ב-JSON מיובאים סטטית ומאוגדים בזמן בנייה.

### Composable useLocale

```typescript
// ui/src/composables/useLocale.ts
const { currentLocale, isRTL, direction, setLocale } = useLocale()
```

- `currentLocale` — ref ריאקטיבי, משותף בין כל מופעי ה-composable
- `isRTL` — מחושב: `true` לעברית
- `direction` — מחושב: `'rtl'` או `'ltr'`
- `setLocale(tag)`:
  1. מעדכן ref `currentLocale`
  2. מגדיר `locale.value` (vue-i18n)
  3. שומר ב-`localStorage`
  4. מגדיר `document.documentElement.lang`
  5. best-effort `PATCH /api/auth/locale` לסנכרון עם צד השרת

### תמיכת RTL

RTL מיושם לכל רכיב (לא על `<html>`) כדי למנוע שבירת פריסת הסרגל הצדדי. רכיבים משתמשים בערך המחושב `direction` כדי להגדיר תכונות `dir` במקומות הנדרשים.

### רכיב LanguagePicker

תפריט נפתח `<NSelect>` של Naive UI בסרגל הצדדי עם 6 אפשרויות המסומנות בכתב המקורי שלהן (למשל "עברית" לעברית). בבחירה, קורא `setLocale()`.

### מבנה locale ב-JSON

אובייקטים מקוננים של מרחבי שמות:

```json
{
  "nav": { "dashboard": "Dashboard", "facts": "Facts", ... },
  "common": { "save": "Save", "cancel": "Cancel", ... },
  "login": { "title": "Login", ... },
  "dashboard": { ... },
  "tasks": { ... },
  "settings": { ... },
  "locale": { "en": "English", "ru": "Русский", "he": "עברית", ... }
}
```

### סנכרון locale עם צד השרת

`PATCH /api/auth/locale` (endpoint לוח בקרה):
1. מאמת locale דרך `i18n.IsSupported()`
2. מעדכן `user_channels.locale` עבור כל ערוצי המשתמש
3. מחזיר פרופיל משתמש מעודכן

## הוספת שפה חדשה

### צד שרת

1. צור `internal/i18n/catalog/<code>.toml` עם כל המפתחות הנדרשים
2. הוסף את `language.Tag` ל-`SupportedLanguages` ב-`resolve.go`
3. הוסף דגל RTL אם נדרש
4. הוסף מיפויי שמות שפה ב-`skill/locale/locale.go` (שמות באנגלית + רוסית)

### צד לקוח

1. צור `ui/src/i18n/locales/<code>.json` עם כל מרחבי השמות
2. ייבא ורשום ב-`ui/src/i18n/index.ts`
3. הוסף ל-RTL_LOCALES ב-`ui/src/composables/useLocale.ts` אם RTL
4. הוסף תווית מקורית בכל קבצי ה-locale תחת `locale.<code>`
