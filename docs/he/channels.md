# ערוצים

Iulita תומך במספר ערוצי תקשורת. כל ערוץ ממיר הודעות ספציפיות לפלטפורמה לפורמט `IncomingMessage` אוניברסלי ומנתב אותן דרך העוזר.

## יכולות ערוצים

כל ערוץ מצהיר על יכולותיו דרך מסכת ביטים בכל הודעה:

| יכולת | Console | Telegram | WebChat |
|------------|---------|----------|---------|
| סטרימינג | דרך bubbletea | כן (מבוסס עריכה) | כן (WebSocket) |
| Markdown | דרך glamour | כן | HTML |
| תגובות | לא | לא | לא |
| כפתורים | לא | כן (inline keyboard) | כן |
| מחוון הקלדה | כן | כן | לא |
| HTML | לא | לא | כן |

יכולות הן לכל הודעה (לא לכל ערוץ), מה שמאפשר התנהגות מעורבת כאשר מספר ערוצים חולקים עוזר אחד. מיומנויות יכולות לבדוק יכולות דרך `channel.CapsFrom(ctx)` כדי להתאים את פורמט הפלט שלהן.

## Console TUI

מצב ברירת המחדל — צ'אט טרמינל במסך מלא מופעל על ידי [bubbletea](https://github.com/charmbracelet/bubbletea).

### תכונות

- **פריסה במסך מלא**: viewport (היסטוריית שיחה) + מפריד + textarea (קלט) + שורת סטטוס
- **עיבוד Markdown**: דרך [glamour](https://github.com/charmbracelet/glamour) עם גלישת מילים אדפטיבית
- **סטרימינג**: הופעת טקסט חיה עם מחוון spinner
- **פקודות**: `/help`, `/status`, `/compact`, `/clear`, `/quit`
- **הנחיות אינטראקטיביות**: אפשרויות ממוספרות לאינטראקציות מיומנויות (למשל, בחירת מיקום למזג אוויר)
- **זיהוי צבע רקע**: מתאים עיבוד לפני הפעלת bubbletea

### ארכיטקטורה

```
tuiModel (bubbletea)
    ├── viewport.Model (היסטוריית שיחה ניתנת לגלילה)
    ├── textarea.Model (קלט משתמש)
    ├── statusBar (שם מיומנות, מוני טוקנים, עלות)
    └── streamBuf (טקסט סטרימינג חי)
```

מבנה `console.Channel` מחזיק `*tea.Program` מוגן על ידי `sync.RWMutex`. תוכנית bubbletea רצה ב-goroutine משלה (חוסם את `Start()`), בעוד `StartStream`, `SendMessage` ו-`NotifyStatus` נקראים מ-goroutine העוזר בו-זמנית.

### גשר סטרימינג

כאשר העוזר מזרים תשובה:

1. `StartStream()` מחזיר סגורים `editFn` ו-`doneFn`
2. `editFn(text)` שולח `streamChunkMsg` ל-bubbletea (טקסט מלא מצטבר)
3. `doneFn(text)` שולח `streamDoneMsg` ל-bubbletea (סיום והוספה להיסטוריה)
4. כל ההודעות הן thread-safe דרך `p.Send()` של bubbletea

### פקודות

| פקודה | תיאור |
|---------|-------------|
| `/help` | הצג את כל הפקודות עם תיאורים |
| `/status` | מוני מיומנויות, עלות יומית, טוקני סשן, מספר הודעות |
| `/compact` | הפעלה ידנית של דחיסת היסטוריה (אסינכרוני) |
| `/clear` | נקה היסטוריית שיחה בזיכרון (TUI בלבד) |
| `/quit` / `/exit` | צא מהאפליקציה |

### דו-קיום עם מצב שרת

במצב קונסולה, השרת רץ ברקע:
- לוגים מופנים ל-`iulita.log` (לא stderr, כדי למנוע השחתת TUI)
- לוח הבקרה עדיין נגיש בכתובת המוגדרת
- Telegram וערוצים אחרים רצים לצד ה-TUI

## Telegram

בוט Telegram מלא עם סטרימינג, debouncing והנחיות אינטראקטיביות.

### הגדרה

1. צור בוט דרך [@BotFather](https://t.me/BotFather)
2. הגדר את הטוקן: `iulita init` (keyring) או משתנה סביבה `IULITA_TELEGRAM_TOKEN`
3. אופציונלי: הגדר `telegram.allowed_ids` להגבלת מזהי משתמשי Telegram ספציפיים

### תכונות

- **רשימה לבנה של משתמשים**: `allowed_ids` מגביל מי יכול לשוחח עם הבוט. ריק = הרשה לכולם (אזהרה נרשמת)
- **debouncing הודעות**: הודעות מהירות מאותו צ'אט מאוחדות (חלון ניתן להגדרה)
- **עריכות סטרימינג**: תשובות מופיעות בהדרגה דרך `EditMessageText` (מוגבל ל-עריכה/1.5 שניות)
- **חלוקת הודעות**: הודעות מעל 4000 תווים מפוצלות בגבולות פסקה/שורה/מילה, תוך שמירה על בלוקי קוד
- **threading תשובות**: הנתח הראשון מגיב להודעת המשתמש; נתחים עוקבים הם עצמאיים
- **מחוון הקלדה**: פעולת `ChatTyping` נשלחת כל 4 שניות בזמן עיבוד
- **ניטור בריאות**: `GetMe()` נקרא כל 60 שניות לזיהוי בעיות קישוריות
- **הנחיות אינטראקטיביות**: מקלדות inline לאינטראקציות מיומנויות (מיקום מזג אוויר וכו')
- **תמיכת מדיה**: תמונות (גודל מקסימלי), מסמכים (מגבלת 30MB), קול/שמע (עם תמלול)
- **פקודות מובנות**: `/clear` (נקה היסטוריה), פקודות רשומות מותאמות

### צינור עיבוד הודעות

```
עדכון Telegram נכנס
    │
    ├── callback query? → ניתוב למטפל הנחיות
    ├── לא הודעה? → דלג
    ├── משתמש לא ברשימה לבנה? → דחה
    ├── פקודת /clear? → טפל ישירות
    ├── פקודה רשומה? → נתב למטפל
    ├── הנחיה פעילה? → נתב טקסט להנחיה
    │
    ▼
בניית IncomingMessage
    │ Caps = Streaming | Markdown | Typing | Buttons
    │
    ├── פתרון משתמש (פלטפורמה → UUID iulita)
    ├── חיפוש locale מ-DB
    ├── הורדת מדיה (תמונה/מסמך/קול)
    ├── בדיקת מגבלת קצב
    │
    ▼
Debounce
    │ איחוד הודעות מהירות (טקסט מחובר עם \n)
    │ טיימר מתאפס בכל הודעה חדשה
    │
    ▼
Handler (Assistant.HandleMessage)
```

### Debouncer

ה-debouncer מאחד הודעות מהירות מאותו צ'אט כדי למנוע קריאות LLM מרובות:

- לכל `chatID` יש buffer עם טיימר `time.AfterFunc`
- הוספת הודעה מאפסת את הטיימר
- כשהטיימר מסתיים, כל ההודעות ב-buffer מאוחדות:
  - טקסט מחובר עם `"\n"`
  - תמונות ומסמכים מחוברים
  - מטא-דאטה של ההודעה הראשונה נשמר
- אם `debounce_window = 0`, הודעות מעובדות מיידית (לא חוסם)
- `flushAll()` מעבד buffers שנותרו בזמן כיבוי

### חלוקת הודעות

תשובות ארוכות מפוצלות לנתחים תואמי Telegram (מקסימום 4000 תווים):

1. ניסיון פיצול בגבולות פסקה (`\n\n`)
2. ניסיון פיצול בגבולות שורה (`\n`)
3. ניסיון פיצול בגבולות מילה (` `)
4. פיצול גולמי כמוצא אחרון
5. **מודעות לבלוקי קוד**: אם מפצלים בתוך בלוק ```, סוגרים עם ``` ופותחים מחדש בנתח הבא

### הגדרות

| מפתח | ברירת מחדל | תיאור |
|-----|---------|-------------|
| `telegram.token` | — | טוקן בוט (ניתן לטעינה חמה) |
| `telegram.allowed_ids` | `[]` | רשימה לבנה של מזהי משתמשים (ריק = הרשה לכולם) |
| `telegram.debounce_window` | 2s | חלון איחוד הודעות |

## WebChat

צ'אט אינטרנט מבוסס WebSocket מוטבע בלוח הבקרה.

### פרוטוקול

**חיבור**: WebSocket ב-`/ws/chat?user_id=<uuid>&username=<name>&chat_id=<optional>`

**הודעות נכנסות** (לקוח → שרת):
```json
{
  "text": "user message",
  "chat_id": "web:abc123",
  "prompt_id": "prompt_123_1",       // רק לתשובות הנחיה
  "prompt_answer": "option_id"       // רק לתשובות הנחיה
}
```

**הודעות יוצאות** (שרת → לקוח):

| סוג | מטרה | שדות מרכזיים |
|------|---------|------------|
| `message` | תשובה רגילה | `text`, `timestamp` |
| `stream_edit` | עדכון סטרימינג | `text`, `message_id`, `timestamp` |
| `stream_done` | סטרימינג הסתיים | `text`, `message_id`, `timestamp` |
| `status` | אירועי עיבוד | `status`, `skill_name`, `success`, `duration_ms` |
| `prompt` | שאלה אינטראקטיבית | `text`, `prompt_id`, `options[]` |

### אימות

WebChat **לא** משתמש ב-UserResolver. הממשק הקדמי משיג טוקן JWT דרך `/api/auth/login`, שולף `user_id` מהנתונים, ומעביר אותו כפרמטר שאילתת WebSocket. הערוץ סומך על `user_id` זה ישירות.

### סריאליזציית כתיבה

כל כתיבות WebSocket עוברות דרך `sync.Mutex` לכל חיבור כדי למנוע panic מכתיבה מקבילית. כל חיבור נרשם במפת `clients[chatID]`.

### הנחיות אינטראקטיביות

הנחיות משתמשות במזהים מבוססי מונה אטומי: `prompt_<timestamp>_<counter>`. השרת שולח הודעת `prompt` עם אפשרויות; הלקוח מגיב עם `prompt_id` ו-`prompt_answer`. הנחיות ממתינות נשמרות ב-`sync.Map` עם timeout.

## מנהל ערוצים

`channelmgr.Manager` מתזמר את כל מופעי הערוצים בזמן ריצה.

### מחזור חיים

- **StartAll**: טוען את כל מופעי הערוצים מ-DB, מפעיל כל אחד ב-goroutine
- **StopInstance**: מבטל context, ממתין על ערוץ done (timeout 5 שניות)
- **AddInstance / UpdateInstance**: למופעים שנוצרו/שונו בלוח הבקרה
- **טעינה חמה**: `UpdateConfigToken(token)` מפעיל מחדש מופעי Telegram שמקורם בהגדרות

### ניתוב הודעות

כאשר העוזר צריך לשלוח הודעה יזומה (תזכורת, heartbeat):

1. חיפוש איזה מופע ערוץ מחזיק את `chatID` דרך DB
2. אם נמצא ורץ, שימוש בשולח של ערוץ זה
3. חלופה: שימוש בערוץ הראשון שרץ

### סוגי ערוצים נתמכים

| סוג | מקור | טעינה חמה |
|------|--------|------------|
| Telegram | הגדרות או DB | טעינה חמה של טוקן |
| WebChat | DB (bootstrap) | — |
| Console | מצב קונסולה בלבד | — |

## פתרון משתמשים

`DBUserResolver` ממפה זהויות פלטפורמה ל-UUID של iulita:

1. חיפוש `user_channels` לפי `(channel_type, channel_user_id)`
2. אם נמצא → החזרת `user.ID` קיים
3. אם לא נמצא ורישום אוטומטי מופעל:
   - יצירת `User` חדש עם סיסמה אקראית ו-`MustChangePass: true`
   - חיבור ערוץ למשתמש
   - החזרת UUID חדש
4. אם לא נמצא ורישום אוטומטי מושבת → דחייה

**locale לכל ערוץ**: לאחר פתרון, `GetChannelLocale(ctx, channelType, channelUserID)` נקרא כדי להגדיר `msg.Locale` מהעדפה השמורה ב-DB.

## אירועי סטטוס

ערוצים מקבלים התראות `StatusEvent` למשוב UX:

| סוג | מתי | שימוש |
|------|------|-----|
| `processing` | הודעה התקבלה, לפני קריאת LLM | הצג "חושב..." |
| `skill_start` | לפני ביצוע מיומנות | הצג שם מיומנות |
| `skill_done` | לאחר ביצוע מיומנות | הצג משך, הצלחה/כישלון |
| `stream_start` | לפני תחילת סטרימינג | הכן ממשק סטרימינג |
| `error` | בשגיאה | הצג הודעת שגיאה |
| `locale_changed` | לאחר מיומנות set_language | עדכן locale של ממשק |
