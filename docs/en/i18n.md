# Internationalization (i18n) and Localization (l10n)

Iulita supports 6 languages across all layers: backend system messages, console TUI, skill outputs, approval vocabulary, and the Vue 3 dashboard frontend.

## Supported Languages

| Code | Language | RTL | Script |
|------|----------|-----|--------|
| `en` | English | No | Latin |
| `ru` | Russian | No | Cyrillic |
| `zh` | Chinese (Mandarin) | No | Hanzi |
| `es` | Spanish | No | Latin |
| `fr` | French | No | Latin |
| `he` | Hebrew | Yes | Hebrew |

## Architecture

```
Backend (Go)                          Frontend (Vue 3)
┌─────────────────────┐              ┌─────────────────────┐
│ i18n Package        │              │ vue-i18n v9         │
│ ├── Bundle          │              │ ├── JSON locales    │
│ ├── TOML catalogs   │              │ ├── useLocale()     │
│ ├── Context locale  │              │ └── LanguagePicker  │
│ └── Approval words  │              └─────────────────────┘
└─────────────────────┘
         ↕                                    ↕
   user_channels.locale              localStorage + PATCH /api/auth/locale
```

## Backend i18n

### Package Structure

```
internal/i18n/
├── i18n.go        # Bundle, Init(), T(), Tl(), Translate()
├── context.go     # WithLocale(), LocaleFrom()
├── resolve.go     # ResolveLocale(), SupportedLanguages, matcher
└── catalog/
    ├── en.toml
    ├── ru.toml
    ├── zh.toml
    ├── es.toml
    ├── fr.toml
    └── he.toml
```

### Initialization

`i18n.Init()` must be called after DB migrations and before the assistant/skills. It:

1. Creates a `go-i18n/v2` bundle with English as default language
2. Registers the TOML unmarshaler
3. Reads all embedded `*.toml` catalogs from `catalog/`
4. Pre-caches `*i18n.Localizer` instances for all supported languages
5. Parses approval words into O(1) lookup tables

### Translation API

```go
// Context-based (reads locale from context)
i18n.T(ctx, "ConsolePlaceholder")
i18n.T(ctx, "WeatherForTitle", map[string]interface{}{"Location": "Moscow"})

// Direct tag-based (no context)
i18n.Tl(language.Russian, "LocaleSwitched", map[string]interface{}{"Language": "Russian"})

// Core translation function
i18n.Translate(tag, msgID, data...)
```

Fallback chain: locale-specific translation → English translation → raw msgID.

### Context Propagation

```go
// Set locale in context
ctx = i18n.WithLocale(ctx, language.Russian)

// Read locale from context
tag := i18n.LocaleFrom(ctx) // returns language.English if not set
```

The locale flows through the entire request lifecycle:

```
user_channels.locale (DB)
    ↓
msg.Locale (IncomingMessage field)
    ↓
i18n.ResolveLocale(msg.Locale, msg.LanguageCode)
    ↓
i18n.WithLocale(ctx, tag)
    ↓
All downstream i18n.T(ctx, ...) calls
```

### Locale Resolution

`ResolveLocale(channelLocale, msgLang)`:

1. Parse `channelLocale` (DB-stored preference, e.g., `"ru"`)
2. Fall back to `msgLang` (platform language, e.g., Telegram's `LanguageCode`)
3. Run through `language.NewMatcher(SupportedLanguages)` for best-fit matching
4. **Priority**: DB-stored locale > platform language > English default

### Catalog Keys

All 6 locales cover these key groups:

| Group | Count | Examples |
|-------|-------|---------|
| Approval flow | 9 | `ApprovalAffirmative`, `ApprovalNegative`, `ApprovalPrompt` |
| Telegram | 7 | `TelegramHistoryCleared`, `TelegramRateLimited` |
| Console TUI | 17 | `ConsolePlaceholder`, `ConsoleThinking`, `ConsoleProcessing` |
| Console commands | 12 | `CommandHelpHeader`, `CommandStatusSkills` |
| Weather | 30+ | WMO weather codes, `WeatherLocationPrompt` |
| Locale skill | 3 | `LocaleSwitched`, `LocaleUnknown` |
| Assistant | 3 | `AssistantLanguageDirective`, `AssistantCompressionPrompt` |
| Error | 1 | `ErrorUnknownSkill` |

### Approval Words

Each locale defines affirmative and negative words for the approval flow:

```toml
# ru.toml
[ApprovalAffirmative]
other = "да,д,ок,подтвердить,подтверждаю,yes,y,ok,confirm"

[ApprovalNegative]
other = "нет,н,отмена,отменить,no,n,cancel"
```

**Cross-language guarantee**: All non-English locales include English words as fallback. Users can type "yes" regardless of their locale setting.

The approval check is O(1) using pre-parsed word sets (`map[string]map[string]bool`).

### LLM Language Directive

When the locale is not English, a language directive is injected into the dynamic system prompt:

```
# Russian:
"Отвечай на русском языке. Язык ответов должен соответствовать предпочтениям пользователя."

# Hebrew:
"השב בעברית. שפת התשובות חייבת להתאים להעדפת המשתמש."
```

The directive is rendered in the target language so the LLM sees instructions matching its expected response language. Section headers (`## Current Time`, `## Remembered Facts`) remain in English — the LLM understands them regardless.

### TOML Catalog Notes

- **Chinese quotes**: Use `「」` brackets instead of `""` in `zh.toml` — the TOML parser can't handle fullwidth quotes inside regular strings
- **Compression prompt**: Kept in English across all locales — the LLM interprets English summarization instructions fine
- **Summary prefix**: Translated (appears directly in chat)

## Per-Channel Locale Storage

Locale is stored per-channel, not per-user:

```sql
user_channels (
    ...
    locale TEXT NOT NULL DEFAULT 'en'
)
```

This allows different languages on different channels (e.g., Russian on Telegram, English on WebChat).

### Storage Operations

- `UpdateChannelLocale(ctx, chatID, locale)` — called by `set_language` skill
- `GetChannelLocale(ctx, channelType, channelUserID)` — called during message processing
- `GetChannelLocaleByChatID(ctx, chatID)` — alternative lookup

### set_language Skill

Users switch language via chat:

```
User: "switch to Russian"
Bot: "Язык интерфейса изменён на русский."  (in Russian)

User: "переключи на иврит"
Bot: "שפת הממשק שונתה לעברית."  (in Hebrew)
```

The skill:
1. Parses language input (BCP-47 code or name in English/Russian)
2. Validates against supported languages
3. Updates `user_channels.locale` in DB
4. Returns confirmation in the **new** language

## Frontend i18n

### Setup

```typescript
// ui/src/i18n/index.ts
const i18n = createI18n({
    legacy: false,  // Composition API mode
    locale: localStorage.getItem('iulita.locale') || navigator.language.split('-')[0] || 'en',
    fallbackLocale: 'en',
    messages: { en, ru, zh, es, fr, he }
})
```

All 6 JSON locale files are statically imported and bundled at build time.

### useLocale Composable

```typescript
// ui/src/composables/useLocale.ts
const { currentLocale, isRTL, direction, setLocale } = useLocale()
```

- `currentLocale` — reactive ref, shared across all composable instances
- `isRTL` — computed: `true` for Hebrew
- `direction` — computed: `'rtl'` or `'ltr'`
- `setLocale(tag)`:
  1. Updates `currentLocale` ref
  2. Sets `locale.value` (vue-i18n)
  3. Saves to `localStorage`
  4. Sets `document.documentElement.lang`
  5. Best-effort `PATCH /api/auth/locale` to sync backend

### RTL Support

RTL is applied per-component (not on `<html>`) to avoid breaking the sidebar layout. Components use the `direction` computed value to set `dir` attributes where needed.

### LanguagePicker Component

A Naive UI `<NSelect>` dropdown in the sidebar with 6 options labeled in their native scripts (e.g., "עברית" for Hebrew). On selection, calls `setLocale()`.

### JSON Locale Structure

Nested namespace objects:

```json
{
  "nav": { "dashboard": "Dashboard", "facts": "Facts", ... },
  "common": { "save": "Save", "cancel": "Cancel", ... },
  "login": { "title": "Login", ... },
  "dashboard": { ... },
  "tasks": { ... },
  "settings": { ... },
  "locale": { "en": "English", "ru": "Русский", "he": "עברית", ... }
}
```

### Backend Locale Sync

`PATCH /api/auth/locale` (dashboard endpoint):
1. Validates locale via `i18n.IsSupported()`
2. Updates `user_channels.locale` for all of the user's channels
3. Returns updated user profile

## Adding a New Language

### Backend

1. Create `internal/i18n/catalog/<code>.toml` with all required keys
2. Add the `language.Tag` to `SupportedLanguages` in `resolve.go`
3. Add RTL flag if needed
4. Add language name mappings in `skill/locale/locale.go` (English + Russian names)

### Frontend

1. Create `ui/src/i18n/locales/<code>.json` with all namespaces
2. Import and register in `ui/src/i18n/index.ts`
3. Add to RTL_LOCALES in `ui/src/composables/useLocale.ts` if RTL
4. Add native label in all locale files under `locale.<code>`
